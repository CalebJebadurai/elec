# India Election Analysis & Prediction Platform

An open-source, interactive platform for analyzing Assembly and Lok Sabha election data across all Indian states and running custom prediction simulations. Built for news agencies, political analysts, pollsters, and election enthusiasts.

> **⚠ Disclaimer:** Predictions generated by this tool are mathematical simulations based on historical patterns and user-adjustable parameters. They are **not** electoral forecasts and should not be cited as such. Actual election outcomes depend on alliance dynamics, candidate selection, campaign strategies, local issues, and many factors not captured here.

---

## Features

### National Dashboard
- **Interactive India Map** — Choropleth with all 36 states/UTs, click to drill into any state
- **Party Strength Charts** — Cross-state seat distribution with year filters
- **State Comparison** — Head-to-head metrics between any two states
- **Election Timeline** — Upcoming elections and historical trends
- **AE/GE Mode Switcher** — Toggle between Assembly and Lok Sabha elections

### Election Data Explorer
- **State Overview** — Seat distribution and vote share trends across all general elections
- **Constituencies** — Detailed history with winner timelines, margin trends, and swing analysis
- **Party Analysis** — Vote share evolution, seat counts, and cross-election comparisons

### Prediction Simulator (Assembly Elections)
- **Anti-Incumbency Slider** — Model the percentage of incumbent vote share that shifts to the opposition
- **Turnout Control** — Adjust expected voter turnout (55–90%)
- **New Party / Third Front** — Simulate a new entrant with:
  - Statewide vote share allocation
  - Affinity-weighted vote redistribution (which parties lose votes)
  - Per-constituency overrides for targeted scenarios
  - Preset profiles based on top parties in each state
- **Real-time Results** — Seat projections, vote share charts, comparison with latest election, and constituency-level flip indicators

### Community Features
- **Save Predictions** — Bookmark slider configurations with title, description, and state context
- **Publish & Share** — Make predictions public for the community to view
- **Like / Dislike** — React to community predictions
- **User Profiles** — Sign up with mobile number, optionally link Google account

### Additional Features
- **API Key Management** — Developer access for programmatic data queries
- **CSV/JSON Export** — Download election data in standard formats
- **Social Sharing** — OG meta images for shared state/constituency links
- **Embeddable Widgets** — ConstituencyCard and SeatProjection for third-party sites
- **Pricing & Subscriptions** — Tiered access via Razorpay integration

## Quick Start

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/) and Docker Compose
- (Optional) [Bun](https://bun.sh/) for local frontend development

### 1. Clone & Configure

```bash
git clone https://github.com/CalebJebadurai/elec.git
cd elec

# Copy environment template and set your values
cp .env.example .env
```

Edit `.env` and set **at minimum**:
```env
POSTGRES_PASSWORD=your_secure_password
DATABASE_URL=postgresql://elec:your_secure_password@db:5432/elec
JWT_SECRET=generate_a_random_64_character_string
```

### 2. Start Development Environment

```bash
docker compose up -d
```

This starts:
- **PostgreSQL** at `localhost:5434` — auto-loads election data from CSV
- **Redis** at `localhost:6379` — response caching
- **FastAPI** at `localhost:8000` — REST API ([docs at /docs](http://localhost:8000/docs))
- **Vite Frontend** at `localhost:5173` — React app with hot reload

### 3. (Optional) Start Jupyter Notebooks

```bash
docker compose --profile dev up jupyter
```

Access at `localhost:8888` with the token from your `.env` file.

### 4. Production Deployment

#### Option A: Split Deploy (Recommended — Free/Cheap)

**Frontend → Vercel (free):** Connect your GitHub repo at vercel.com.

**API + Database → Railway ($5–15/mo):** Add PostgreSQL plugin, connect repo, set env vars.

**Seed the database** (first time only):
```bash
./infra/seed-railway-all.sh "postgresql://postgres:xxx@xxx.railway.app:5432/railway"
```

#### Option B: Single VPS with Docker Compose ($6–12/mo)

```bash
git clone https://github.com/CalebJebadurai/elec.git && cd elec
cp .env.example .env  # Edit with production values
FRONTEND_TARGET=prod docker compose --profile prod up -d
```

#### Option C: GCP 3-Tier Architecture (Production-Grade HA)

See `infra/terraform/` for full Terraform configs.

```bash
cd infra && ./setup.sh your-gcp-project-id
cd terraform && cp terraform.tfvars.example terraform.tfvars
terraform plan && terraform apply
```

## API Reference

All endpoints accept `?state=State_Name` and `?election_type=AE|GE` where applicable.

### Election Data

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/states` | List all states |
| `GET` | `/elections` | Paginated with year/party/district filters |
| `GET` | `/elections/{id}` | Single election record |
| `GET` | `/elections/{year}/results` | Winners for a year |
| `GET` | `/years` | Election years with counts |
| `GET` | `/parties` | Party list with vote totals |
| `GET` | `/constituencies` | Constituencies for a state |
| `GET` | `/districts` | Districts with constituency counts |
| `GET` | `/candidates?name=...` | Search candidates |
| `GET` | `/stats/summary` | Aggregate statistics |
| `GET` | `/swings/constituency/{name}` | Year-by-year results |
| `GET` | `/swings/state` | State-level party swing |
| `GET` | `/swings/constituencies` | All swing histories |
| `GET` | `/predict/data` | Prediction dataset |

### National Dashboard

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/national/state-summary` | All states with latest stats |
| `GET` | `/national/party-strength` | Cross-state seat distribution |
| `GET` | `/national/turnout-trends` | Historical turnout by state |
| `GET` | `/national/upcoming-elections` | Next scheduled elections |
| `GET` | `/national/compare` | Head-to-head state comparison |
| `GET` | `/national/party-map` | Party seats across states |

### Auth

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/auth/verify-otp` | Verify Firebase phone OTP, get JWT |
| `POST` | `/auth/google-link` | Link Google account |

| `GET` | `/auth/me` | Get current user |
| `PUT` | `/auth/me` | Update profile |
| `GET` | `/auth/users/me/data` | Export personal data |
| `DELETE` | `/auth/users/me` | Delete account |

### Bookmarks & Community

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/bookmarks` | List own bookmarks |
| `GET` | `/bookmarks/public?sort=recent\|popular` | Browse public predictions |
| `POST` | `/bookmarks` | Save prediction |
| `PUT` | `/bookmarks/{id}` | Update bookmark |
| `DELETE` | `/bookmarks/{id}` | Delete bookmark |

### Export, API Keys, Payments, OG, Admin

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `POSTGRES_PASSWORD` | Yes | Database password |
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `JWT_SECRET` | Yes | JWT signing secret (min 32 chars) |
| `ALLOWED_ORIGINS` | No | CORS origins (default: `http://localhost:5173`) |
| `REDIS_URL` | No | Redis URL (default: `redis://redis:6379`) |
| `FIREBASE_PROJECT_ID` | For auth | Firebase project |
| `FIREBASE_API_KEY` | For auth | Firebase API key |
| `GOOGLE_CLIENT_ID` | For Google link | Google OAuth client ID |
| `RAZORPAY_KEY_ID` | For payments | Razorpay API key |
| `RAZORPAY_KEY_SECRET` | For payments | Razorpay secret |
| `RAZORPAY_WEBHOOK_SECRET` | For payments | Razorpay webhook secret |
| `SENTRY_DSN` | For monitoring | Sentry DSN |
| `DISABLE_ADMIN` | For production | Set `true` to disable admin routes |
| `JUPYTER_TOKEN` | For dev | Jupyter access token |
