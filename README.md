# Tamil Nadu Election Analysis & Prediction Platform

An open-source, interactive platform for analyzing Tamil Nadu Assembly Election data (1971–2021) and running custom 2026 prediction simulations. Built for news agencies, political analysts, pollsters, and election enthusiasts.

> **⚠ Disclaimer:** Predictions generated by this tool are mathematical simulations based on historical patterns and user-adjustable parameters. They are **not** electoral forecasts and should not be cited as such. Actual election outcomes depend on alliance dynamics, candidate selection, campaign strategies, local issues, and many factors not captured here.

---

## Features

### Election Data Explorer
- **State Overview** — Seat distribution and vote share trends across 12 general elections
- **234 Constituencies** — Detailed history with winner timelines, margin trends, and swing analysis
- **Party Analysis** — Vote share evolution, seat counts, and cross-election comparisons

### 2026 Prediction Simulator
- **Anti-Incumbency Slider** — Model the percentage of incumbent vote share that shifts to the opposition
- **Turnout Control** — Adjust expected voter turnout (55–90%)
- **New Party / Third Front** — Simulate a new entrant (e.g., TVK, MNM) with:
  - Statewide vote share allocation
  - Affinity-weighted vote redistribution (which parties lose votes)
  - Per-constituency overrides for targeted scenarios
  - Preset profiles: DMK-leaning, ADMK-leaning, Neutral disruptor
- **Real-time Results** — Seat projections, vote share charts, 2021 comparison, and constituency-level flip indicators

### Community Features
- **Save Predictions** — Bookmark your slider configurations with title and description
- **Publish & Share** — Make predictions public for the community to view
- **Like / Dislike** — React to community predictions (no comments to prevent toxicity)
- **User Profiles** — Sign up with mobile number, optionally link Google account

## Quick Start

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/) and Docker Compose
- (Optional) [Bun](https://bun.sh/) for local frontend development

### 1. Clone & Configure

```bash
git clone https://github.com/cnickson/elec.git
cd elec

# Copy environment template and set your values
cp .env.example .env
```

Edit `.env` and set **at minimum**:
```env
POSTGRES_PASSWORD=your_secure_password
DATABASE_URL=postgresql://elec:your_secure_password@db:5432/elec
JWT_SECRET=generate_a_random_64_character_string
```

### 2. Start Development Environment

```bash
docker compose up -d
```

This starts:
- **PostgreSQL** at `localhost:5434` — auto-loads election data from CSV
- **FastAPI** at `localhost:8000` — REST API ([docs at /docs](http://localhost:8000/docs))
- **Vite Frontend** at `localhost:5173` — React app with hot reload

### 3. (Optional) Start Jupyter Notebooks

```bash
docker compose --profile dev up jupyter
```

Access at `localhost:8888` with the token from your `.env` file.

## Prediction Engine

The prediction engine runs entirely client-side for instant feedback. The algorithm:

### Baseline Generation
1. Scale 2021 elector counts by growth factor (2026 total / 2021 total)
2. Apply expected turnout to get projected valid votes
3. Apply anti-incumbency: incumbent loses X% of their vote share
   - 70% redistributed to runner-up
   - 30% distributed equally among other parties
4. Normalize vote shares to sum to 100%

### New Party Simulation
When a new party is added:
1. Allocate statewide vote share (or per-constituency overrides)
2. Subtract from existing parties using affinity-weighted redistribution:
   ```
   loss_i = new_votes × (α_i × V_i) / Σ(α_j × V_j)
   ```
   Where `α_i` = affinity weight, `V_i` = party's current votes
3. Recompute winners based on adjusted vote counts

### Caveats
- Based on 2021 results only (post-delimitation constituencies)
- Does not account for: alliance changes, NOTA, candidate-level factors
- Anti-incumbency model is simplified (uniform across all constituencies)
- New party vote distribution is not geographically modeled

## Legal & Compliance

### Election Laws
- This tool complies with the **Representation of the People Act, 1951** (India)
- Predictions are clearly labeled as simulations, not forecasts
- No exit poll data is used or displayed during election periods
- The tool does not attempt to influence voters — it's an analytical/educational instrument

### Data Privacy
- Only mobile number is collected for authentication
- Google account linking is optional (for settings sync)
- No personal data is sold or shared with third parties
- Users can delete their accounts and all associated data

### Political Neutrality
- The prediction engine is mathematical and unbiased
- No party is favored in default parameters
- Historical data is presented without editorial commentary
- Affinity presets are labeled as analytical proxies, not endorsements
